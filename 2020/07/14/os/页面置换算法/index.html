页面置换算法

发表于 2020-07-14
字数统计: 1k | 阅读时长 ≈ 3

好的页面置换算法应有较低的页面更换频率，也就是应将以后不会再访问或长时间不会访问的页面先调出。

最佳（OPT）置换算法

该算法选择的被淘汰页面是以后永不使用，或是在最长时间内不再被访问的页面，以便获得最低的缺页率。然而，由于人们无法预知进程在内存下的若干页面哪个是未来最长时间内不再访问的。因此该算法无法实现。

但可以用最佳置换算法来评价其它算法。

先进先出（FIFO）页面置换算法

优先淘汰最早进入内存的页面。该算法实现简单，只需要把调入内存的页面根据先后次序排成队列。但该算法与进程实际运行时的规律不适应。

FIFO算法还会产生所分配的物理块数增大而缺页率却增大的异常现象，被称为Belady异常。而只有FIFO算法会出现这种异常。

最近最久未使用（LRU）置换算法

选择最近最长时间未访问过的页面予以淘汰。该算法为每一个页面设置一个访问字段，来记录页面自上次被访问以来所经历的时间，选择该字段值最大的页面进行淘汰。

LRU算法性能较好，但需要寄存器和栈的硬件支持。

时钟（CLOCK）置换算法

LRU算法的性能接近OPT算法，但实现起来较为困难，开销较大。FIFO实现简单，但性能较差。因此，操作系统的设计者尝试了许多算法试图以较小的开销来接近LRU的性能，这类算法都是CLOCK算法的变体。因为算法要循环扫描缓冲区，像时钟的指针一样转动，所以称为CLOCK算法。

  • 简单的CLOCK算法给每帧关联一个附加位，称为使用位。当某页首次装入内存时，将该帧的使用位置1。
  • 当该页随后再被访问到时，其使用位也置为1。
  • 对于页替换算法，用于替换的候选帧集合可视为一个循环缓冲区，并有一个指针与之相关联。
  • 当某一页被替换时，该指针被设置成指向缓冲区中的下一帧。
  • 当需要替换一页时，通过扫描缓冲区以查找使用位被置为0的一帧，每当遇到一个使用位为1的帧时，将其置为0。
  • 若在这个操作开始时，所有帧的使用位均为0，则选择遇到的第一个帧替换。
  • 如果所有帧都为1，指针循环扫描一周后所有帧都被置为0，所以可以正常替换。

通过增加使用的位数目，可以使CLOCK算法更加高效。在使用位的基础上增加一个修改位，则得到改进型的CLOCK置换算法。每帧都是处于以下4种情况之一。

  1. 最近未被访问，也未被修改。(u=0, m=0)
  2. 最近被访问，但违背修改。（u=1, m=0）
  3. 最近未被访问，但被修改。(u=0, m=1)
  4. 最近被访问，被修改。（u=1, m=1）

算法执行步骤如下

  1. 从指针当前位置开始，扫描帧缓冲区。在扫描过程中对使用位不做任何修改。选择第一个遇到的(u=0, m=0)用于替换。
  2. 若第1步失败，则重新扫描，查找(u=0, m=1)的帧，选择第一个遇到的这样的帧进行替换。扫描过程中对于每个跳过的帧将使用位置为0。
  3. 若第2步失败。则指针回到最初位置，且集合中所有使用位均为0。重复第一步，如果有必要，重复第二步，直到找到可替换的帧。

改进型CLOCK算法的优点在于替换时首选没有变化的页。由于修改过的页在被替换前必须写回，所以会节省时间。
